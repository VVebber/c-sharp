Null-условия

Null-условные операторы в C# - это специальный синтаксис, который позволяет упростить проверку на null перед вызовом метода или свойства объекта. Они позволяют избежать ошибок NullReferenceException, которые могут возникнуть при попытке обращения к null-объекту.

Null-условные операторы имеют следующий вид: "?." и "??". Рассмотрим каждый из них подробнее.

Оператор "?." (Null-условный оператор):

Оператор "?." используется для вызова методов или свойств объекта, но только в том случае, если объект не является null. Если объект null, то вызов метода или свойства будет проигнорирован и вернется null.

Вот пример, который показывает, как использовать Null-условный оператор:

1

string str = null; int? length = str?.Length; Console.WriteLine(length); // Выведет null

В этом примере мы создали переменную str и присвоили ей значение null. Затем мы использовали Null-условный оператор "?." для вызова метода Length объекта str. Поскольку str равно null, выражение str?.Length вернет null и присвоит его переменной length.

Оператор "??" (Null-сохраняющий оператор):

Оператор "??" используется для определения значения, которое следует использовать, если объект является null. Если объект не является null, то оператор "??" будет проигнорирован и вернется значение объекта.

Рассмотрим пример:

1

string str1 = null; string str2 = str1 ?? "Значение по умолчанию"; Console.WriteLine(str2); // Выведет "Значение по умолчанию"

В этом примере мы создали переменную str1 и присвоили ей значение null. Затем мы использовали Null-сохраняющий оператор "??" для определения значения переменной str2. Если str1 равно null, то значение по умолчанию ("Значение по умолчанию") будет присвоено переменной str2.

Кроме того, можно использовать оператор "??" вместе с "?." для более удобной проверки на null. Рассмотрим пример:

1

string str = null; int length = str?.Length ?? 0; Console.WriteLine(length); // Выведет 0

В этом примере мы использовали Null-условный оператор "?." для вызова метода Length объекта str. Если str равно null, выражение str?.Length вернет null, но мы использовали оператор "??" для установки значения по умолчанию 0. Таким образом, если объект str равен null, то переменная length будет равна 0.

В заключение, Null-условные операторы в C#позволяют упростить проверку на null и избежать ошибок NullReferenceException. Они делают код более читабельным и удобным для работы.

Оператор "?." может использоваться с любым объектом, а оператор "??" может использоваться с любым типом данных, включая примитивные типы, строки и пользовательские классы.

Операторы "?." и "??" могут быть использованы вместе для выполнения более сложной проверки на null. Например, в следующем примере мы используем оба оператора вместе:

1

Person person = null; string firstName = person?.Name?.First ?? "John";

В этом примере мы создали переменную person, которая равна null, и использовали Null-условный оператор "?." для получения свойства Name объекта person. Если person равно null или свойство Name равно null, то выражение person?.Name?.First вернет null. Мы использовали оператор "??" для установки значения по умолчанию "John", которое будет присвоено переменной firstName.

Важно помнить, что Null-условные операторы не являются панацеей от всех ошибок, связанных с null. Некоторые ошибки могут возникнуть, если вы не сможете определить правильное поведение в случае, если объект равен null.

Также стоит помнить, что использование Null-условных операторов может ухудшить производительность кода, особенно если они используются внутри циклов или методов, которые вызываются множество раз. Поэтому следует использовать их с умом и только там, где это действительно необходимо.
